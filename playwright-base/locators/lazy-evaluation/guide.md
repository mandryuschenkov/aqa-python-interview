# Ленивые локаторы и перерисовка DOM (Locator vs ElementHandle)

> [!WARNING]
> **Вопросы (Middle):**
> Перед вами два теста, взаимодействующих с веб-страницей. Между поиском элемента и кликом по нему происходит перерисовка DOM-дерева (эмуляция поведения современных SPA-фреймворков, таких как React или Vue).
> 
> 1. Какой из этих тестов завершится успешно, а какой упадет с ошибкой? Объясните причину такого поведения.
> 2. В чем фундаментальная разница между `query_selector` и `locator` в архитектуре Playwright?

### Ответ:
Первый тест (`test_submit_form_with_element_handle`) завершится ошибкой, так как элемент будет отсоединен от DOM-дерева. Второй тест (`test_submit_form_with_locator`) выполнится успешно.

### Пояснение:

1. **Проблема первого теста (`ElementHandle`):**
   Метод `query_selector` выполняется мгновенно и возвращает указатель на конкретный узел DOM (`ElementHandle`). Когда скрипт `page.evaluate` перерисовывает кнопку, старый узел удаляется из документа. При попытке вызвать `submit_btn.click()` Playwright обнаружит, что сохраненный элемент больше не привязан к DOM (аналог `StaleElementReferenceException` в Selenium), и выбросит исключение.

2. **Успех второго теста (`Locator`):**
   Метод `locator` не ищет элемент в момент вызова. Он создает "рецепт" (инструкцию) для поиска элемента по заданному селектору. Поиск и взаимодействие происходят только в момент вызова действия (`click()`). К этому моменту DOM уже перерисован, `locator` найдет новую кнопку и успешно выполнит клик.

3. **Архитектурная разница (к вопросу №2):**
   * **`query_selector` (Устаревший подход):** Мгновенное вычисление, отсутствие встроенных механизмов ожидания, подверженность ошибкам при динамических изменениях DOM.
   * **`locator` (Рекомендуемый подход):** Ленивое вычисление, строгий режим по умолчанию, автоматическое ожидание готовности элемента (видимость, отсутствие перекрытия, стабильность) перед выполнением действия.

### Навыки:
Понимание механизма ленивого вычисления локаторов в Playwright, знание концепции авто-ожиданий и умение избегать проблем с динамически обновляемым DOM, характерных для современных веб-приложений.
