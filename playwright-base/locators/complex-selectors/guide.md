# Сложные XPath и CSS селекторы: функции, оси навигации и комбинирование

> 🟡 **Вопросы (Middle):**
> Перед вами одна тестовая функция `test_complex_selectors`, которая содержит 27 последовательных действий с использованием сложных XPath или CSS-селекторов для взаимодействия с элементами HTML-страницы.
>
> 1. Опишите, что делает каждый из селекторов в тесте `test_complex_selectors` — действия #1 — #27.

## Пояснения

1. **[XPATH] `//li[not(@class="menu-item active")]/a`** — click. Поиск элемента `<a>`, являющегося дочерним по отношению к `<li>`, у которого атрибут `class` НЕ равен `"menu-item active"`. Функция `not()` инвертирует условие. Полезный паттерн для выбора всех элементов, кроме выделенных/активных.

2. **[CSS] `.form-input[type="email"]`** — fill. Селектор комбинирует классовый селектор `.form-input` и атрибутивный `[type="email"]`. Классовый селектор ищет элемент, у которого атрибут `class` **содержит** `form-input` (не точное совпадение, а частичное — элемент может иметь несколько классов через пробел). Атрибутивный селектор `[type="email"]` фильтрует по точному значению `type`. Метод `.fill()` заполняет поле текстом.

3. **[XPATH] `//label[text()="Phone"]`** — click. Поиск элемента `<label>` с точным текстовым содержимым "Phone". Функция `text()` сравнивает весь текстовый узел элемента. В отличие от `contains(text(), ...)`, требует точного совпадения без дополнительных пробелов.

4. **[CSS] `a[href$="terms"]`** — click. Атрибутивный CSS-селектор с оператором `$=` находит ссылки, у которых `href` заканчивается на "terms". Найдёт `<a href="https://company.com/terms">`.

5. **[XPATH] `//input[contains(@id, "user")]`** — fill. Поиск элемента `<input>`, у которого атрибут `id` содержит подстроку "user" в любом месте. Функция `contains()` выполняет частичное совпадение. Найдёт `<input id="username">`, так как "username" содержит "user".

6. **[CSS] `input[class*="form-input"][type="tel"]`** — fill. Комбинирование двух атрибутивных селекторов: `class*="form-input"` ищет подстроку "form-input" в атрибуте class, `[type="tel"]` фильтрует по точному значению. Множественные селекторы без пробела означают, что все условия должны выполняться на одном элементе.

7. **[XPATH] `//a[starts-with(@href, "https://")]`** — click. Поиск элемента `<a>`, у которого атрибут `href` начинается с "https://". 

8. **[CSS] `.form-group:nth-of-type(3) label`** — click. Псевдокласс `:nth-of-type(3)` выбирает третий элемент `<div class="form-group">` среди своих братьев того же типа.

9. **[XPATH] `//label[text()="Phone"]/parent::div/input`** — fill. Цепочка из трёх шагов: находим `<label>` с точным текстом "Phone", поднимаемся вверх через ось `parent::` к `<div class="form-group">`, затем спускаемся к дочернему `<input>`. 

10. **[CSS] `tr.order-row:nth-child(3) td:nth-child(2)`** — click. Комбинированный CSS-селектор: `tr.order-row:nth-child(3)` выбирает третий дочерний элемент `<tr>` внутри `<tbody>` с классом `order-row`, `td:nth-child(2)` выбирает вторую ячейку внутри этой строки. `:nth-child()` считает все дочерние элементы независимо от класса.

11. **[XPATH] `//label[text()="Username"]/following-sibling::input`** — fill. Находим `<label>` с текстом "Username", затем через ось `following-sibling::` переходим к сестринскому элементу `<input>`, который расположен **после** текущего элемента `<label>` и имеет того же родителя. Ось `following-sibling::` выбирает все сестринские элементы-соседи, идущие после текущего в DOM-дереве (не включая сам текущий элемент).

12. **[CSS] `td.status-badge.status-pending`** — click. Комбинирование двух классов в одном селекторе. Находит `<td>`, у которого одновременно есть оба класса: `status-badge` и `status-pending`. CSS не имеет оператора "и" — запись классов подряд без пробелов означает, что все они должны быть на одном элементе.

13. **[XPATH] `//input[@id="email"]/preceding-sibling::label`** — click. Находим `<input>` с `id="email"`, затем через ось `preceding-sibling::` переходим к сестринскому элементу `<label>`, который расположен **перед** текущим элементом `<input>` и имеет того же родителя. Ось `preceding-sibling::` выбирает все сестринские элементы-соседи, идущие перед текущим в DOM-дереве (не включая сам текущий элемент).

14. **[CSS] `#user-panel .form-label`** — click. Пробел выбирает все элементы `.form-label`, являющиеся потомками элемента с `id="user-panel"`. Включает потомков на любом уровне вложенности.

15. **[XPATH] `//input[@id="phone"]/ancestor::div[@class="form-group"]`** — click. Находим `<input>` с `id="phone"`, затем через ось `ancestor::` поднимаемся ко всем предкам и фильтруем по `<div class="form-group">`. Ось `ancestor::` включает родительский элемент и всех предков вверх по дереву.

16. **[CSS] `.menu > .menu-item > a`** — click last. Child combinator `>` выбирает элементы только на первом уровне вложенности. `<a>` должен находиться на первом уровне вложенности внутри `.menu-item`, который в свою очередь должен находиться на первом уровне вложенности внутри `.menu`. Метод `.last()` выбирает последний элемент из коллекции.

17. **[XPATH] `//li[@data-role="nav-item"][not(@class="menu-item active")]/a`** — click. Множественные условия в скобках: `[@data-role="nav-item"]` проверяет атрибут, `[not(@class="menu-item active")]` исключает активные элементы. Два условия в одном predicated-блоке работают как логическое И (`AND`).

18. **[CSS] `[data-testid="main"] .panel-heading`** — click. Селектор состоит из двух частей: `[data-testid="main"]` — атрибутивный селектор, находит элемент с атрибутом `data-testid` и точным значением `"main"`; `.panel-heading` — классовый селектор, ищет элемент, у которого атрибут `class` **содержит** `panel-heading`. Между ними — descendant combinator (пробел), который выбирает все элементы `.panel-heading`, находящиеся внутри элемента с `data-testid="main"` на любом уровне вложенности.

19. **[XPATH] `//li[@data-role="nav-item"][not(@class="menu-item active")]/a[contains(@href, "settings")]`** — click. Многоступенчатый XPath-селектор с тремя условиями в двух predicated-блоках. Разбор по шагам: (1) ищем любой элемент `<li>` в документе; (2) первый predicate `[@data-role="nav-item"]` фильтрует — атрибут `data-role` должен быть равен `"nav-item"` (точное совпадение); (3) второй predicate `[not(@class="menu-item active")]` исключает элементы, у которых `class` равен `"menu-item active"` — функция `not()` инвертирует условие, аналогично `@class != "menu-item active"`; (4) `/a` спускаемся к дочернему элементу `<a>`; (5) третий predicate `[contains(@href, "settings")]` применяемый к `<a>` — атрибут `href` должен содержать подстроку `"settings"` в любом месте (например, `"https://app.com/settings/profile"` или `"/admin/settings"`). Два predicated-блока на `<li>` работают как логическое И (`AND`) — оба условия должны выполняться одновременно.

20. **[CSS] `input[type="text"][placeholder="Enter username"]`** — fill. Множественные атрибутивные селекторы. Элемент `<input>` должен иметь одновременно `type="text"` И `placeholder="Enter username"`. Каждый дополнительный селектор `[...]` сужает выборку. (Примечание: в HTML-разметке `placeholder="Enter username"` находится у `<input id="username">`, а не у `<input id="email">`, поэтому селектор находит именно поле Username.)

21. **[CSS] `tr[data-status="pending"] td:nth-child(2)`** — click. Комбинирование data-атрибута с `:nth-child()`. Сначала находим `<tr>` с атрибутом `data-status="pending"`, затем второго `<td>` внутри него. Полезный паттерн для выбора конкретных ячеек в таблицах.

22. **[CSS] `a[href*="company"]`** — click. Атрибутивный селектор `*=` ищет подстроку "company" в любом месте значения `href`. В отличие от `^=` и `$=`, `*=` не привязан к началу или концу строки. Аналог в XPath: `//a[contains(@href, "company")]`.

23. **[CSS] `input[placeholder^="Enter"]`** — fill (nth(1)). Атрибутивный селектор `^=` с методом `.nth(1)` для выбора второго элемента из коллекции. Находит все `<input>` с placeholder, начинающимся на "Enter", и выбирает второй. Playwright использует 0-based индексацию: `.nth(0)` — первый, `.nth(1)` — второй.

24. **[CSS] `a[href$="privacy"]`** — click. Атрибутивный селектор `$=` ищет ссылки, у которых `href` заканчивается на "privacy". Найдёт `<a href="https://company.com/privacy">`. Аналог в XPath: `//a[ends-with(@href, "privacy")]` (XPath 2.0+).

25. **[CSS] `input[type="text"][class*="form-input"]`** — fill. Множественные атрибутивные селекторы: `type="text"` (точное совпадение) + `class*="form-input"` (содержит подстроку). Элемент должен удовлетворять обоим условиям одновременно.

26. **[XPATH] `//div[@class="form-group"]/input[1]`** — fill. XPath-индекс **[1] применяется к конкретному элементу в пути**, а не к результату всей выборки. Разбор: (1) `//div[@class="form-group"]` находит все `<div>` с `class="form-group"`; (2) `/input[1]` для каждого найденного `<div>` выбирает только первый дочерний `<input>`. Индекс [1] внутри скобок после `<input>` означает «первый `<input>` внутри своего родителя». Если у `<div>` три `<input>`, будет выбран только первый из них.

27. **[XPATH] `(//div/input)[1]`** — fill. XPath-индекс **[1] применяется к результату всей выборки в скобках**. Разбор: (1) `(//div/input)` — сначала выполняем весь запрос `//div/input` и собираем ВСЕ `<input>` внутри любых `<div>` в единую коллекцию; (2) `[1]` выбирает только первый элемент из этой коллекции. В отличие от #26, где индекс фильтровал внутри каждого родителя, здесь индекс выбирает один элемент из глобального результата. Пример: если найдено 3 `<input>` в разных `<div>`, `//div/input[1]` вернёт по первому из каждого `<div>` (3 элемента), а `(//div/input)[1]` вернёт только один первый элемент из всей коллекции.

### Навыки:
- **XPath функции:** `not()`, `text()`, `contains()`, `starts-with()` — для точного, частичного и контекстного поиска элементов
- **XPath оси навигации:** `parent::` (родитель), `following-sibling::` (следующие братья), `preceding-sibling::` (предыдущие братья), `ancestor::` (все предки) — для навигации по DOM-дереву
- **XPath множественные условия:** `[@attr="val"][not(...)]` — несколько predicated-блоков работают как логическое И
- **CSS атрибутивные операторы:** `^=` (начинается с), `$=` (заканчивается на), `*=` (содержит подстроку) — для поиска по части значения атрибута
- **CSS псевдоклассы:** `:nth-of-type()` (n-й элемент своего типа), `:nth-child()` (n-й дочерний элемент) — для выбора элемента по позиции
- **CSS комбинаторы:** descendant (пробел), child (`>`), комбинирование классов и атрибутов — для построения иерархических селекторов
- **Умение выбирать между XPath и CSS:** XPath лучше для навигации по DOM (оси, текст), CSS — для иерархии и стилизации
