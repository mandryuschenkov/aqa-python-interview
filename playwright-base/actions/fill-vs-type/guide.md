# Ввод текста: fill vs type

> [!WARNING]
> **Вопросы (Middle):**
> Перед вами два теста, которые заполняют строку поиска для вызова виджета автодополнения. В первом тесте используется метод `fill`, во втором — метод `type` с искусственной задержкой.
> 
> 1. В чем фундаментальная разница между методами `fill` и `type` на уровне событий браузера?
> 2. Какой из этих методов официально признан устаревшим в новых версиях Playwright и какой современный метод пришел ему на смену?
> 3. Приведите примеры специфических сценариев, когда метод `fill` может не справиться с задачей заполнения формы.

### Ответ:
Метод `fill` вставляет текст целиком, генерируя только события изменения значения. Метод `type` эмулирует физическое нажатие клавиш. Метод `type` устарел, вместо него используется `press_sequentially`.

### Пояснение:

1. **Разница на уровне событий:**
   * **`fill`:** Работает атомарно. Он предварительно очищает поле и мгновенно устанавливает новое значение, генерируя только события `input` и `change`. Это самый быстрый и надежный способ ввода.
   * **`type`:** Эмулирует реальную клавиатуру. Он не очищает поле перед вводом, а последовательно генерирует события `keydown`, `keypress`, `keyup` и `input` для каждого отдельного символа.

2. **Устаревание метода `type`:**
   Кандидат должен знать, что метод `type` официально признан устаревшим. Разработчики Playwright заменили его на метод `press_sequentially`.

3. **Сценарии обязательного использования посимвольного ввода:**
   Хотя `fill` рекомендуется использовать почти всегда, посимвольный ввод (`press_sequentially`) жизненно необходим в следующих ситуациях:
   * **Сложные JavaScript-маски:** Поля для ввода номера телефона, кредитной карты или одноразового пароля, где логика фронтенда жестко перехватывает события `keydown` для форматирования текста на лету.
   * **Виджеты автодополнения:** Строки поиска, которые отправляют сетевые запросы только после паузы в наборе текста или реагируют исключительно на событие отпускания клавиши. В таких случаях используется параметр задержки между нажатиями.

### Навыки:
Глубокое понимание того, как фреймворк взаимодействует с браузером под капотом, знание актуального API Playwright (отказ от устаревших методов) и умение выбирать правильный инструмент в зависимости от реализации фронтенда.
