# ClassVar в Pydantic BaseModel

> 🟡 **Вопросы (Middle):**
> Перед вами Pydantic модель с константой `FIELD_LABELS`, помеченной как `ClassVar`.
> 
> 1. Зачем `FIELD_LABELS` объявлена через `ClassVar`, а не как обычное поле модели?
> 2. Что произойдёт, если убрать `ClassVar` и оставить только аннотацию `dict[str, str]`?
> 3. Какое поведение у `model_dump()` и `model_fields` в данном коде?
> 4. Можно ли получить доступ к `FIELD_LABELS` через экземпляр класса?

### Ответ:
1. `ClassVar` исключает атрибут из обработки как поля модели.
2. Без `ClassVar` Pydantic v2 воспринимает атрибут как обычное поле с дефолтным значением.
3. С `ClassVar` — `model_dump()` и `model_fields` не включают `FIELD_LABELS`; без `ClassVar` — включают.
4. Да, доступ через экземпляр возможен — `ClassVar` не блокирует чтение на уровне runtime.

### Пояснение:
1. **Зачем ClassVar:** `typing.ClassVar` — стандартный способ пометить атрибут как константу класса (PEP 526). Pydantic v2 сканирует аннотации и при обнаружении `ClassVar` исключает атрибут из `model_fields`, `__init__` и сериализации.

2. **Без ClassVar:** Pydantic v2 определяет поля по аннотации типа + присвоенного значения на уровне класса. `FIELD_LABELS: dict[str, str] = {...}` — классический синтаксис аннотированного атрибута. Pydantic создаёт поле модели с дефолтным значением. Ошибки не будет — в отличие от случая аннотации без значения, когда Pydantic не может определить дефолт.

3. **model_dump / model_fields:** С `ClassVar` — `model_dump()` и `model_fields` не включают `FIELD_LABELS`. Без `ClassVar` — оба включают: `model_dump()` добавит `FIELD_LABELS` в выходной словарь, `model_fields` будет содержать четыре ключа. Это влияет на JSON-схему и валидацию.

4. **Доступ через экземпляр:** `ClassVar` — метаданные для type checker и фреймворков. На уровне runtime это обычный атрибут класса. `model.FIELD_LABELS` работает через `__getattribute__`. Type checker может выдать предупреждение при доступе через экземпляр.

### Навыки:
* Понимание различия между полями модели и класс-константами в Pydantic.
* Знание типа `ClassVar` и его влияния на поведение фреймворков.
* Умение предсказывать результат `model_dump()` и `model_fields`.
