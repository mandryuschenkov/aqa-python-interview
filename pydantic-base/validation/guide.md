# Проверка базовых знаний по валидации Pydantic

> Расскажите как пройдёт валидация при выполнении метода `model_validate`.

Для класса `Client`:
1. Поле `id` отсутствует в `client_dict`, но для него указан аргумент `default_factory=uuid4`. Ошибки валидации нет.
2. Поля `first_name` и `last_name` будут провалидированы без ошибок.
3. Поле `middle_name` указано в классе `Client` и оно обязательное, поэтому будет ошибка валидации.
4. Значение ключа `"is_active"` будет преобразовано в `True` без ошибки валидации.
5. Значение ключа `"age"` будет преобразовано в `18` без ошибки валидации.
6. В словаре `client_dict` есть ключ `gender`, который не объявлен как поле в классе `Client`, поэтому он будет проигнорирован без ошибки.
7. В поле `status` будет ошибка при валидации так как `Literal` проверяет точное совпадение значения с одним из перечисленных вариантов.

Для вложенного класса `Contact`:
1. В поле `phone` не будет ошибки валидации.
2. В значении поля `email` отсутствует точка перед `com`. Будет ошибка при валидации типа `EmailStr`.
3. В поле `telegram` ошибки при валидации не будет, так как указано значение `None` по умолчанию.


> Задайте дополнительные вопросы последовательно:
> * Как отменить приведение типов и вызвать ошибку при валидации полей?
> * Примените для полей `is_active` и `age` разные способы вызвать ошибку валидации.
> * Как вызвать ошибку при валидации необъявленных ключей, например для `gender`?

1. Есть несколько вариантов:
   * использовать `model_config = ConfigDict(strict=True)` для строгих проверок на все поля класса
   * использовать для отдельных полей Strict-классы, например `StrictBool`, `StrictInt`
   * использовать класс `Strict` как дополнительный в аннотации типа
2. Для ключа `is_active` можно выбрать класс `StrictBool`, а для `age` можно добавить класс `Strict` в аннотацию типа:
   ```python
   is_active: StrictBool = True
   age: Annotated[int, Strict()] = Field(..., ge=18, le=80)
   ```
3. Нужно добавить в конфиг аргумент `extra="forbid"`:
   ```python
   class Client(BaseModel):
       model_config = ConfigDict(extra="forbid")
   ```
