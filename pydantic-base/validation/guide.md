# Проверка базовых знаний по валидации Pydantic

> 🟡 **Вопрос (Middle):**
> Перед вами Pydantic модель и словарь с данными. Расскажите пошагово, как пройдёт валидация при выполнении метода `model_validate(client_dict)`. Какие поля вызовут ошибки, а какие будут обработаны успешно?

### Ответ:
Ошибки валидации возникнут в полях `middle_name` (обязательное, но отсутствует), `status` (значение не из списка `Literal`) и `email` (невалидный формат адреса вложенной модели). Остальные поля будут успешно провалидированы или приведены к нужному типу.

### Пояснение:

**Для класса `Client`:**
1. Поле `id`: отсутствует в словаре, но имеет `default_factory=uuid4`. Ошибки не будет.
2. Поля `first_name` и `last_name`: провалидируются без ошибок.
3. Поле `middle_name`: объявлено как обязательное (нет значения по умолчанию), но его нет в словаре. **Будет ошибка валидации об отсутствии обязательного поля.**
4. Поле `is_active`: значение `"is_active"` будет автоматически приведено к `True` без ошибки.
5. Поле `age`: строка `"age"` (если там число) будет приведена к типу `int` без ошибки.
6. Поле `gender`: присутствует в словаре, но не объявлено в модели. Будет просто проигнорировано (по умолчанию `extra="ignore"`).
7. Поле `status`: значение не совпадает с вариантами из `Literal`. **Будет ошибка валидации.**

**Для вложенного класса `Contact`:**
1. Поле `phone`: провалидируется без ошибок.
2. Поле `email`: в значении отсутствует точка перед `com`. **Будет ошибка валидации (неверный формат EmailStr).**
3. Поле `telegram`: отсутствует в словаре, но имеет значение по умолчанию `None`. Ошибки не будет.

> 🔴 **Дополнительные вопросы (Senior):**
> 1. Как отменить неявное приведение типов и вызывать ошибку при несовпадении типов?
> 2. Покажите разные способы сделать строгую проверку только для полей `is_active` и `age`.
> 3. Как заставить Pydantic выдавать ошибку при наличии в словаре необъявленных ключей (например, `gender`)?

### Объяснение к дополнительным вопросам:

1. **Глобальная строгая валидация:**
   Кандидат должен упомянуть настройку конфига `model_config = ConfigDict(strict=True)`.

2. **Точечная строгая валидация:**
   * Использование специальных Strict-типов из библиотеки: `is_active: StrictBool = True`.
   * Использование аннотаций: `age: Annotated[int, Strict()] = Field(...)`.

3. **Запрет лишних полей:**
   Нужно изменить настройку `extra` в конфигурации модели:
   ```python
   class Client(BaseModel):
       model_config = ConfigDict(extra="forbid")
   ```

### Навыки:
Умение кандидата предсказывать поведение Pydantic при сериализации грязных данных. Знание особенностей неявного приведения типов, работы с `Literal`, вложенными моделями и конфигурацией `ConfigDict`.
