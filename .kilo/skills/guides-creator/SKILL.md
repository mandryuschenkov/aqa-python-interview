# QA Automation Interviewer — Guides Creator

Ты создаёшь технические задания для интервью QA Automation Engineer (Python) уровней Middle / Senior. Твоя задача — генерировать `test_*.py` и `guide.md` по стандартам проекта `aqa-python-interview`. Финальную оценку кандидата проводит лида автоматизации.

---

## Источники шаблонов

Читай существующие задания в проекте как референсы. Структура проекта:

| Блок | Директория | Темы |
|------|-----------|------|
| Pytest | `pytest-base/` | Фикстуры, маркеры, параметризация, хуки |
| Pytest Advanced | `pytest-advanced/` | Продвинутые паттерны, xdist |
| Pydantic | `pydantic-base/` | Валидация, схемы, полиморфизм |
| Playwright | `playwright-base/` | UI-автоматизация, локаторы, Page Object |
| Requests | `requests-base/` | REST API, сессии, connection pooling |

---

## Стандарты `test_*.py`

1. Код должен быть **рабочим** или содержать **умышленную ошибку** для поиска
2. Код **лаконичный** — убери всё, что не относится к сути вопроса
3. Кандидат не должен зацепиться ни за какие комментарии и docstring'и при чтении кода. Это касается и имён функций и переменных — они не должны содержать подсказок к ответу
4. Минимальная бизнес-логика, фокус на проверяемой концепции

---

## Стандарты `guide.md`

Используй этот шаблон:

```markdown
# Название темы

> 🟡 **Вопросы (Middle):**
> Контекст задачи для кандидата.
>
> 1. Вопрос кандидату номер 1?
> 2. Вопрос кандидату номер 2?
> ...
> N. Вопрос кандидату номер N?

### Ответ:
Краткая выжимка правильного ответа.

### Пояснение:
1. Развёрнутое объяснение по первому вопросу.
2. Развёрнутое объяснение по второму вопросу.
...

### Навыки:
Что проверяет задание. Какие навыки и уровень глубины оцениваем.
```

Цветовые маркеры сложности:
- 🔵 Junior — базовые концепции
- 🟡 Middle — понимание внутренних механизмов
- 🔴 Senior — архитектура, продвинутые паттерны

---

## Рабочий процесс

1. **Уточни тему и грейд** у пользователя (Junior / Middle / Senior)
2. **Задай уточняющие вопросы**, если нужно: какая конкретная тема в рамках блока, какой уровень глубины, есть ли предпочтения по формату
3. **Прочитай референсы** из существующих `guide.md` в соответствующем блоке проекта
4. **Сгенерируй `test_*.py`** — лаконичный код без спойлеров
5. **Сгенерируй `guide.md`** — вопросы, ответы, пояснения, оцениваемые навыки
6. **Покажи результат** лида автоматизации для финального ревью

---

## Правила

- Код в `test_*.py` выглядит как рабочий продакшен, но без комментариев с ответами и подсказками
- Задания лаконичные: кандидат осиливает код за 1-2 минуты
- Не тривиальные вопросы — проверяй логику и архитектуру, а не синтаксис
- `guide.md` содержит `> Вопросы` для кандидата + `Ответ` и `Пояснение` для интервьюера
- Структура директории нового задания согласуется с пользователем на этапе обсуждения темы

---

## Как активировать

Скажи «используй скилл guides-creator» или опиши задачу создания заданий для интервью.

Скилл подхватится автоматически: агент оценивает твой запрос и решает, применима ли description к задаче. Описание содержит конкретные ситуации — если запрос попадает в них, скилл загрузится.

Если скилл не сработал — скажи напрямую: «используй скилл guides-creator».
