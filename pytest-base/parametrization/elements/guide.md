# Оценка знаний базовой параметризации в Pytest

> 🟡 **Вопросы (Middle):**
> Перед вами класс с двумя параметризованными тестами.
> 
> * Опишите, как работает этот код и сколько всего тестов будет сгенерировано.
> * Расскажите о недостатках текущего подхода к параметризации.
> * Предложите улучшения (как сделать код более читаемым, поддерживаемым и информативным).

### Ответ:
Будет сгенерировано 5 тестов (3 из первого декоратора и 2 из второго). Текущий подход имеет плохую читаемость логов (отсутствуют `ids`), использует хардкод и передает словарь целиком, что усложняет отладку.

### Пояснение:

1. **Как это работает:**
   Используется декоратор `@pytest.mark.parametrize` для запуска одной и той же тестовой логики с разными входными данными.

2. **Недостатки параметризации:**
   * **Плохая читаемость имен тестов.** Pytest сгенерирует имена по умолчанию (например, `test_user_processing[user0]`, `test_is_adult[17-False]`). По этим именам в отчетах (например, Allure) будет сложно понять бизнес-смысл проверяемого кейса.
   * **Захардкоженные данные.** Значения переданы списком прямо в декоратор, что раздувает тестовый файл при большом количестве кейсов.
   * **Передача словарей целиком (во втором тесте).** Передача сложных структур (словарей/объектов) напрямую через параметр делает логи Pytest нечитаемыми и усложняет отладку.
   * **Отсутствие аннотаций типов** у аргументов тестовой функции.

3. **Предложения по улучшению:**
   * **Добавить `ids` в параметризацию.** Либо передать список строковых описаний (например, `ids=["underage", "exact_18", "adult"]`), либо написать функцию-генератор для параметра `ids`. Либо использовать `pytest.param(17, False, id="underage")`.
   * **Вынести тестовые данные в константы или отдельные файлы** (например, `TEST_USERS = [...]`), чтобы не засорять код самого теста.
   * **Использовать распаковку данных.** Во втором тесте лучше параметризовать `name` и `age` отдельно: `@pytest.mark.parametrize("name, age", [("Alice", 25), ...])`, чем передавать словарь. Это улучшит отображение параметров в отчетах.
   * **Использовать фикстуры с параметризацией (`indirect=True`)**, если подготовка объекта (словаря) требует вычислений.
   * **Вынести тестируемую функцию `is_adult`** за пределы тестовой функции.

### Навыки:
Умение находить проблемы в автотестах (код-ревью). Знание базовых механизмов настройки параметризации в Pytest (таких как `ids` и `pytest.param`) для улучшения читаемости отчетов. Понимание правильной архитектуры тестовых данных.
