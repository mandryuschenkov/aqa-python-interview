# Оценка умения читать автотест, видеть слабые места и предлагать улучшения

> Предложите сразу несколько действий:
> * Опишите всё что видите в этом коде.
> * Расскажите о его недостатках.
> * Предложите улучшения для этого кода.

1. Кандидат сразу должен сказать, что это автотест для фреймворка `pytest`. В нём есть тестовый класс и тестовая функция.
2. Также он должен увидеть библиотеку `requests` и сказать, что этот автотест тестирует API.
3. В самой тестовой функции кандидат должен отметить где находится сам вызов метода API, а также две проверки: 
   * на статус код 200
   * на наличие ключа `uuid` внутри объекта `resp.json()`
4. В недостатках вы должны услышать:
   * нет описания тест-кейса в docstring
   * неполное название переменной `response`
   * полный URL в вызове метода API без вынесения `BASE_URL` в переменные окружения
   * нет проверки типа для объекта `resp.json()` и поэтому проверка `assert "uuid" in resp.json()` может привести к ложноположительному результату автотеста
   * отсутствие логирования ответа API перед проверками
   * отсутствие интеграции автотеста с какой-либо TMS
5. Для улучшения данного автотеста кандидат должен предложить:
   * написать полное имя переменной `response`
   * добавить проверку типа `response.json()` и формата UUID
   * переписать проверку `response.json()` в идеале через Pydantic, но и если через JSON-схему и библиотеку `jsonschema` - тоже хорошо
   * переместить вызов метода API в отдельный хелпер
   * реализовать фикстуру для настройки хелпера API перед самим автотестом
   * добавить логирование ответа API
   * добавить интеграцию с TMS
